# Мультимодальный Telegram-бот

Учебный проект. Бот отвечает на вопросы, генерирует изображения, отслеживает курсы валют и криптовалют.

## Возможности

- Текстовые ответы (YandexGPT)
- Генерация изображений (локальный сервер, модель bk-sdm-small)
- Курсы валют (freecurrencyapi) и криптовалют (CoinGecko)
- Установка ценовых целей с уведомлениями
- Опросы через FSM

## Установка

1. Клонировать репозиторий
2. Установить зависимости: `pip install -r requirements.txt`
3. Создать `.env` с ключами (см. `.env.example`)
4. Запустить сервер генерации: `python server/run_local.py --device cpu`
5. Запустить бота: `python main.py`

Схема работы бота

Схема 1: Полный маршрут сообщения (общая картина)

ПОЛЬЗОВАТЕЛЬ
    │
    ▼
1. Telegram → [main.py] → диспетчер (dp)
    │
    ▼
2. [middlewares/ratelimit.py] → проверка спама (если часто — отсекаем)
    │
    ▼
3. [handlers/multimodal.py] → главный хендлер (ловит все сообщения, кроме команд)
    │
    ├─ проверка: "пользователь в опросе?" → если да → сообщаем и выходим
    │
    ├─ проверка лимита запросов (check_rate_limit)
    │
    ▼
4. [services/classifier.py] → определяем НАМЕРЕНИЕ
    │
    ├─ "currency" → передаём в [handlers/currency.py] → ответ сразу
    ├─ "crypto"   → передаём в [handlers/currency.py] → ответ сразу
    ├─ "generate" → идём дальше
    └─ "other"    → идём дальше
    │
    ▼
5. [services/dialog_service.py] → отправляем запрос в YandexGPT
    │
    ├─ добавляем системный промпт (с данными poll_data)
    ├─ добавляем историю диалога
    ├─ отправляем → получаем JSON-массив:
    │     [{"text": "..."}, {"image": "..."}]
    │
    ▼
6. [handlers/multimodal.py] → чистим JSON, парсим в список задач
    │
    ▼
7. [services/chain.py] → запускаем ЦЕПОЧКУ ОБЯЗАННОСТЕЙ
    │
    ├─ TextExecutor → видит {"text": ...} → отправляет мгновенно
    ├─ ImageExecutor → видит {"image": ...} → ставит в очередь
    │
    ▼
8. [services/queue.py] → ОЧЕРЕДЬ (воркеры в фоне)
    │
    ├─ Воркер забирает задачу
    ├─ вызывает [services/api/fusionbrain_api.py] → генерирует картинку
    ├─ отправляет результат пользователю
    │
    ▼
ПОЛЬЗОВАТЕЛЬ (получает ответ)

8	Пользователь получает ответ	—
Итог: Текст → мгновенно. Очередь не используется.

Схема 3: Пример 2 — пользователь пишет «нарисуй корабль»

Итог: Бот ответил мгновенно → картинка пришла отдельно через 3 минуты.

Схема 4: Пример 3 — пользователь пишет «курс доллара»

Базовые вопросы (что делает бот)
1. Что умеет твой бот?
«Мой бот — мультимодальный ассистент. Он умеет:

Отвечать на текстовые вопросы через YandexGPT

Генерировать картинки по запросу через локальный сервер

Показывать курсы валют и криптовалют

Устанавливать цели для отслеживания курса с уведомлениями

Проводить опросы и запоминать данные пользователя

Менять стиль общения по запросу (например, говорить как хиппи)

Показывать статистику по диалогам и список активных целей»

2. Как бот генерирует картинки?
«Бот отправляет промпт в локальный сервер, который запущен на FastAPI. Сервер использует модель Stable Diffusion (bk-sdm-small) и генерирует картинку размером 512×512. Генерация занимает 2–3 минуты, поэтому бот ставит задачу в очередь и сразу отвечает пользователю, что картинка в очереди. Когда картинка готова — она приходит отдельным сообщением.»

3. Как бот обрабатывает валюты и криптовалюты?
«Для валют я использую freecurrencyapi.com, для криптовалют — CoinGecko. Бот извлекает из текста код валюты или криптовалюты, получает курс и отправляет ответ. Если пользователь говорит «следить за биткоином» — запускается FSM: бот просит ввести целевую цену, затем показывает кнопки подтверждения.»

Архитектурные вопросы (как устроен бот)
4. Какая архитектура у бота?
«Бот построен по трёхслойной архитектуре:

Presentation layer — handlers/ (обрабатывают сообщения и команды)

Business logic layer — services/ (вся логика: классификация, диалог, цепочка, очередь, уведомления)

Data access layer — models/ и database.py (хранение пользователей, контекста, целей)
Это позволяет легко расширять бота и поддерживать код в чистоте.»

5. Что такое цепочка обязанностей и зачем она?
«Цепочка обязанностей — это паттерн, при котором задача передаётся по цепочке обработчиков, пока один из них не обработает её. В моём проекте цепочка состоит из TextExecutor и ImageExecutor. TextExecutor обрабатывает текстовые задачи, ImageExecutor — задачи на генерацию картинок. Если задача не текст — TextExecutor передаёт её дальше. Это позволяет легко добавлять новые типы задач, не меняя существующий код.»

6. Зачем нужна асинхронная очередь?
«Очередь нужна для долгих операций — в моём случае это генерация картинок. Вместо того чтобы ждать 2–3 минуты и блокировать бота, я ставлю задачу в очередь и сразу отвечаю пользователю. Воркеры в фоне забирают задачи и генерируют картинки. Это позволяет боту оставаться отзывчивым, даже когда несколько пользователей одновременно запрашивают картинки.»

7. Как работает персонализация?
«Пользователь проходит опрос через команду /poll — отвечает на 4 вопроса: имя, возраст, город, деятельность. Данные сохраняются в poll_data в модели пользователя. При каждом запросе к YandexGPT данные подставляются в системный промпт. Благодаря этому бот обращается к пользователю по имени и адаптирует ответы под его профиль. При команде /clean данные опроса очищаются вместе с историей диалога.»

8. Как бот определяет, что хочет пользователь?
«Бот использует гибридный подход к классификации: сначала проверяет текст по словарям (быстро и дёшево). Если находит ключевые слова (биткоин, доллар, нарисуй) — сразу определяет намерение. Если не находит — отправляет запрос в YandexGPT-классификатор, который возвращает одно из намерений: currency, crypto, generate или `other».»

Технические вопросы (детали реализации)
9. Как хранятся данные пользователей?
«Все пользователи хранятся в словаре users_db в памяти. Периодически они сохраняются в файл db_backup.json. У каждого пользователя есть: системный промпт, история диалога (до 20 сообщений), список целей и данные опроса. Для защиты от состояния гонки я использую asyncio.Lock при работе с целями.»

10. Как работает dialog_service.py?
«Этот сервис формирует запрос к YandexGPT. Он берёт системный промпт (с инструкцией вернуть JSON-массив), добавляет данные пользователя из опроса, затем добавляет историю диалога из database.py и отправляет запрос. Ответ от YandexGPT сохраняется в контекст и возвращается в multimodal.py для дальнейшей обработки.»

11. Как бот обрабатывает JSON от YandexGPT?
«YandexGPT возвращает JSON-массив с ключами text и image. Например: [{"text": "..."}, {"image": "..."}]. В multimodal.py я очищаю JSON от лишнего текста с помощью _clean_ai_response(), парсю его в список задач и передаю каждую задачу в цепочку обязанностей. Текст идёт в TextExecutor, картинка — в ImageExecutor (и далее в очередь).»

12. Как происходит фоновое уведомление о достижении цели?
«Фоновый сервис alert_checker запускается при старте бота и работает в бесконечном цикле. Каждую минуту он проверяет все активные цели пользователей. Если курс валюты или криптовалюты достиг целевого значения, он деактивирует цель и отправляет пользователю уведомление. При завершении бота сервис останавливается через stop_event.»

13. Почему промпты для картинок на английском?
«Модель bk-sdm-small лучше понимает промпты на английском языке. Поэтому в системном промпте YandexGPT я явно указываю, что промпты для изображений должны быть на английском, а текстовые ответы — на русском.»

14. Как защищён бот от спама?
«Я использую middleware RateLimitMiddleware, который ограничивает частоту сообщений. По умолчанию — одно сообщение в секунду. Если пользователь отправляет сообщения чаще, бот просит подождать и не передаёт сообщение дальше.»

Если спросят про слабые места
15. Какие есть ограничения?
«Основные ограничения:

Генерация картинок занимает 2–3 минуты на процессоре

Бот обрабатывает только одну цель за раз (нельзя установить цель на биткоин и доллар в одном сообщении)

Классификатор может ошибаться на сложных запросах

Данные хранятся в памяти, при перезапуске загружаются из JSON — это не масштабируется для больших нагрузок
В коммерческой версии я бы использовал базу данных (PostgreSQL) и вынес бы генерацию в отдельный микросервис.»
